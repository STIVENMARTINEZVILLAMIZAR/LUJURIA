# 📱 EroticLiveX iOS App

Native iOS application for the EroticLiveX platform, built with **SwiftUI** and integrated with **Firebase** for authentication, real-time database, cloud messaging, and more.

## 🎯 Overview

EroticLiveX iOS is a full-featured streaming platform app that allows creators and viewers to connect, stream live content, buy/sell items in the marketplace, and communicate in real-time.

### Platform Requirements
- **iOS Deployment Target:** 14.0+
- **Development:** Xcode 15+ on macOS
- **Language:** Swift 5.9+
- **Architecture:** MVVM (Model-View-ViewModel)

### Key Technologies
- **UI Framework:** SwiftUI
- **Backend:** Firebase (Auth, Firestore, Storage, Messaging)
- **Async:** async/await with Combine
- **Dependency Management:** CocoaPods

## 🚀 Quick Start

### 1. Install Dependencies
```bash
cd ios
pod install
```

### 2. Open Workspace
```bash
open EroticLiveX.xcworkspace
```

### 3. Run on Simulator
```bash
# In Xcode: Select simulator → Cmd + R
# Or via CLI:
xcodebuild -workspace ios/EroticLiveX.xcworkspace \
  -scheme EroticLiveX \
  -configuration Debug \
  -derivedDataPath ios/build
```

### 4. Test Login
- Launch app and see login screen
- Enter test credentials
- Tap "Registrarse" to create account
- View main app after authentication

## 🏗️ Architecture

### MVVM Pattern
```
View (SwiftUI) 
  ↓
ViewModel (@ObservedObject)
  ↓
Service (Manager class)
  ↓
Firebase Backend
```

### Example: Authentication Flow
1. **AuthView** displays login form
2. User enters credentials
3. **AuthView** calls `FirebaseAuthManager.signIn()`
4. **FirebaseAuthManager** makes Firebase API call
5. Auth state listener fires in **AppDelegate**
6. `@Published` properties update
7. **ContentView** detects auth state change
8. **MainTabView** displayed

## 🔄 Data Flow

### Real-time Updates

// Listen to user data changes

    // Update @Published properties

## 🆘 Troubleshooting

### Pod Installation Issues
```bash
pod repo update
rm -rf Pods
rm Podfile.lock
pod install
```

### Build Failures
1. Clean: **Cmd + Shift + K**
2. Close Xcode
3. Delete derived data: `rm -rf ~/Library/Developer/Xcode/DerivedData/*`
4. Reopen workspace
5. Build again

### Firebase Not Initializing
- Verify `GoogleService-Info.plist` is in Xcode project
- Check **Build Phases** → **Copy Bundle Resources** includes it
- Confirm Bundle ID matches Firebase project

### Notifications Not Working
- Enable "Remote Notifications" in Capabilities
- Upload APNs certificate to Firebase Console
- Verify device is not on WiFi (if testing on device)
